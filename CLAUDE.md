# Брандмауэр — Project Overview

## Концепт

**Одна фраза:** на поле горит огонь, он расползается каждый ход — ставь блоки-стенки, чтобы его запереть.

**Ключевая эмоция:** не "красиво уложил", а "хладнокровно отдал угол огню и выиграл". Фигуры неудобные — идеальную стену не построишь. Игрок всегда стоит перед выбором: спасти дальний угол или закрыть центр с тремя домами. Правильное решение — сознательно пожертвовать малым.

**Партия:** 2–4 минуты.

---

## Механика

**Поле:** сетка 8×8. Состояния клеток: `0` трава, `1` стена, `2` огонь, `3` выгоревшее.

**Огонь как фронт — 3 визуальных состояния:**
- `2` 🔥 — где огонь горит сейчас (активный фронт)
- `3` чёрная клетка — где огонь был в прошлый ход (выгорело)
- превью (пульсирующая точка) — куда огонь перекинется в следующий ход

После хода огня текущий фронт (`2`) выгорает в `3`, а новые клетки загораются в `2`. Геймплейно идентично «расползающемуся блобу» (блоб всё равно расширялся только по периметру), меняется лишь вид внутренностей.

**Ход:**
1. Игрок берёт одну фигуру из трея и ставит на свободные клетки → они становятся стеной
2. Огонь расползается: фронт поджигает 4 соседей (↑↓←→), затем сам выгорает. Стены и выгоревшее не горят
3. Повтор

**Трей:** в состоянии 6 фигур, но видны только **3 активные** (очередь скрыта). После использования фигуры очередь сдвигается, добавляется новая случайная.

**Победа:** огонь заперт (add.length === 0 после хода огня). Очки = спасённые дома × 100 + (40 − ходов) × 10.

**Поражение:** огонь дошёл до любого дома 🏠.

## Структура кода (game.js)

| Блок | Что делает |
|---|---|
| `showPage(n)` | Навигация между страницами 1→2→3 |
| `LEVELS[]` | 9 конфигов уровней (seeds, houses, minDist, zone) |
| `pickLevel(idx)` | Переход на страницу 3 + запуск уровня |
| `start(lvIdx)` | Генерация поля: расставляет очаги и дома по правилам уровня |
| `buildBoard()` | Создаёт 64 div-ячейки в #board |
| `onBoardTap` | Постановка в 2 тапа: 1-й тап — превью, 2-й в ту же клетку — ставит |
| `placeAim()` | Ставит подтверждённую фигуру из `S.ghost`, обновляет трей, запускает огонь |
| `anchorFor(r,c)` | Центрирует фигуру под пальцем (offset по центру bounding box) |
| `spreadFire()` | Распространяет огонь, проверяет победу/поражение |
| `nextBurn()` | Возвращает Set клеток, которые загорятся следующим ходом |
| `render()` | Перерисовывает доску (классы CSS на каждой ячейке) |
| `renderTray()` | Перерисовывает трей с мини-превью фигур |
| `popupWin/Lose()` | Показывает финальный попап через #ovl |

## Уровни

```
Лёгкие (1–3): 1 очаг в углу (zone:corner), 2–3 дома, дистанция 4–5
Средние (4–6): 1–2 очага у края (zone:edge), 3–4 дома, дистанция 4–5
Сложные (7–9): 2–3 очага у одного края — широкий фронт, 4–5 домов, дистанция 4–5
```

**Балланс (важно — гарантия «спасти все дома»):** дистанция огонь→дом держится 4–5
на всех уровнях. Очаги ставятся только в углу или у края — НЕ в центре. Центральный
очаг расползается ромбом во все 4 стороны и требует полного окружения кольцом
(~4 клетки/ход против роста периметра, с кривыми фигурами почти нереально) — поэтому
центр убран. Угловой/краевой огонь идёт фронтом в одну сторону, его реально
перегородить линией стен. Очаги одного уровня кучкуются на одной стороне (общий якорь
`side`+`anchor`) → единый фронт.

Параметры каждого уровня в `LEVELS[]` в `game.js`:
- `seeds` — количество очагов
- `houses` — количество домов
- `minDist` — минимальное расстояние (Manhattan) от очага до дома
- `zone` — `'corner'` = очаг в углу, `'edge'` = очаги у случайного края (один фронт)

## Структура страниц

**Страница 1** — название + правила → кнопка "Выбрать уровень"  
**Страница 2** — сетка 3×3 уровней (Легко / Средне / Сложно)

**Попап победы:** "Поздравляю, прошёл!" + счёт + кнопки (след. уровень / ещё раз / к уровням)  
**Попап поражения:** "Дом сгорел" + кнопки (попробовать снова / к уровням)

## Важные детали реализации

- `elementFromPoint(e.clientX, e.clientY)` — надёжное определение ячейки под пальцем на мобиле (не `pointerenter`)
- События доски добавляются **один раз** (не внутри `buildBoard`), иначе дублируются при рестарте
- Ghost-фигура центрируется через `centerOffset(shape)` → `[maxY>>1, maxX>>1]`
- **Постановка в 2 тапа** (`onBoardTap`): 1-й тап ставит `S.aim`+`S.ghost` (превью остаётся видимым после того, как убрал палец), 2-й тап в ту же клетку (`S.aim`) подтверждает. Тап в другую клетку — переносит превью. Это решает проблему «палец закрывает обзор»
- `S.aim` — клетка, по которой целятся; сбрасывается при выборе др. фигуры; поворот (⟳) перерисовывает превью на том же `S.aim`
